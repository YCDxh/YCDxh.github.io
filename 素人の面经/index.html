数据库

1. SQL 操作对比

操作类型 语法 特点 事务日志 恢复性
DELETE DELETE FROM table WHERE… 逐行删除，可回滚 记录完整日志 可恢复
TRUNCATE TRUNCATE TABLE table 立即释放数据页 记录页释放操作 不可回滚
DROP DROP TABLE table 删除表结构 无日志记录 不可恢复

2. 索引优化策略

B+树索引结构

索引失效场景

  1. 对索引列进行数学运算或函数处理
  2. 使用前导通配符的LIKE查询（如 ‘%value’）
  3. 隐式类型转换（如字符串列使用数字查询）
  4. OR条件中存在非索引列
  5. 违反最左前缀原则的联合索引查询

3. 事务隔离级别对比

隔离级别 脏读 不可重复读 幻读 锁机制
READ UNCOMMITTED ✔️ ✔️ ✔️ 无锁
READ COMMITTED ✖️ ✔️ ✔️ 行级写锁
REPEATABLE READ ✖️ ✖️ ✔️ 间隙锁
SERIALIZABLE ✖️ ✖️ ✖️ 表级锁

4. EXPLAIN 执行计划详解

字段 说明 优化建议
type 访问类型（const > ref > range > index > ALL） 尽量达到range级别以上
key 实际使用的索引 检查是否使用最优索引
rows 预估扫描行数 超过1000行需优化
Extra 附加信息（Using filesort/Using temporary需要优化） 避免出现filesort和temporary

5. MVCC 实现机制

版本链结构

Read View 组成要素：

  1. m_ids：当前活跃事务ID列表
  2. min_trx_id：最小活跃事务ID
  3. max_trx_id：预分配的下个事务ID
  4. creator_trx_id：创建该Read View的事务ID

6. 日志系统对比

日志类型 存储引擎 记录内容 用途 写入方式
redo log InnoDB 物理日志（页修改） 崩溃恢复 顺序写
undo log InnoDB 逻辑日志（SQL逆向操作） 事务回滚 随机写
binlog Server层 逻辑日志（SQL语句） 主从复制 追加写

7. 锁机制进阶

行锁类型对比

锁类型 兼容性 使用场景
记录锁（Record Lock） 互斥 精确匹配唯一索引
间隙锁（Gap Lock） 互斥 防止幻读，锁定范围区间
临键锁（Next-Key Lock） 互斥 记录锁+间隙锁组合
插入意向锁 兼容 插入操作前的意向锁

2. 垃圾回收算法对比

算法 优点 缺点 适用场景

3. Spring 事务传播机制

传播行为 说明 使用场景
REQUIRED 支持当前事务，不存在则新建 默认选择
REQUIRES_NEW 新建独立事务 审计日志
NESTED 嵌套事务 部分回滚需求
SUPPORTS 有事务则加入，没有则不使用 查询操作

4. Redis 持久化对比

方式 原理 优点 缺点
RDB 定时快照 恢复快、体积小 可能丢失数据
AOF 追加日志 数据安全 文件大、恢复慢
混合 RDB+AOF 平衡优缺点 需要Redis4+
