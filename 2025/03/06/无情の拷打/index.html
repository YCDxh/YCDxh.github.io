一、前后端交互与HTTP通信

  1. 跨域问题
    • 触发条件：协议、域名、端口任意一个不同。

    • 解决方案：
    ◦ CORS（跨域资源共享）：后端设置响应头Access-Control-Allow-Origin。

    ◦ 代理服务器：前端通过Nginx或Node.js代理转发请求。

    ◦ JSONP（仅限GET请求）。

  2. GET vs POST
    • GET：数据通过URL传输，有长度限制，适合非敏感数据。

    • POST：数据在请求体中传输，更安全，适合敏感数据（需结合HTTPS）。

  3. Spring MVC参数获取方式

  • @PathVariable：获取URL路径参数，如/user/{id}。
  • @RequestParam：获取查询参数，如/user?name=John。
  • @RequestBody：解析JSON/XML请求体到Java对象。
  • HttpServletRequest：直接操作请求头、Cookie等原始数据。
  1. HTTP加密与安全
    • 传输加密：使用HTTPS（SSL/TLS）加密整个通信过程。
    • 敏感数据存储：
    • 数据库：加密存储（如AES算法）。
    • 传输：避免明文传输，使用哈希（如密码加盐）或非对称加密。

二、Java基础

  1. 集合框架

    • ArrayList：基于动态数组，随机访问快（O(1)），增删慢（O(n)）。

    • LinkedList：基于双向链表，增删快（O(1)），随机访问慢（O(n)）。

    • Map接口：

      ◦ HashMap：无序，允许null键/值，线程不安全。

      ◦ LinkedHashMap：保持插入顺序。

      ◦ ConcurrentHashMap：线程安全，分段锁机制。

  2. OOP特性

    • 封装：隐藏内部实现，通过getter/setter访问数据。
    • 继承：extends实现类继承，implements实现接口。
    • 多态：同一方法在不同对象中有不同行为（重写、重载）。

  3. 异常处理

    • throw：手动抛出异常（如throw new IllegalArgumentException()）。
    • throws：声明方法可能抛出的异常（如public void readFile() throws IOException）。

  4. 基础类型与包装类

    • 基础类型：int, double, boolean等，直接存储值。
    • 包装类：Integer, Double, Boolean等，支持null和泛型（如集合中必须用包装类）。

三、Spring框架

  1. IOC（控制反转）

    • 核心思想：由Spring容器管理Bean的创建和依赖注入。
    • 应用优势：
    • 对象之间的耦合度或者说依赖程度降低。
    • 资源变的容易管理。

  2. AOP（面向切面编程）

    • 核心概念：切面（Aspect）、切入点（Pointcut）、通知（Advice）。
    • 应用优势：
      ◦ 将日志记录从核心业务逻辑中分离出来

    ◦ 一行注解即可实现日志记录。

    • 项目应用：
      ◦ 日志记录：在方法执行前后打印日志。

      ◦ 事务管理：通过@Transactional注解管理数据库事务。

  1. Spring MVC结构

    • DAO层：数据访问（如JPA、MyBatis操作数据库）。
    • Service层：业务逻辑处理。
    • Controller层：接收HTTP请求，返回响应。
    • VO/DTO：数据传输对象，用于层间解耦。

  2. @Autowired原理
    • 基于类型自动注入，若有多个实现类需配合@Qualifier指定名称。

• 方法级别：定义具体请求路径，与类路径拼接形成完整 URL。

3. @PathVariable 注解

作用：从 URL 路径中提取动态参数，绑定到方法参数。
• 适用场景：RESTful API 中通过路径标识资源（如 /users/{id}）。
• 支持类型：基本类型（int）、包装类（Long）、字符串、UUID、自定义类型（需转换器）。

示例：

请求示例：
• URL：/person/profile/123/John/true
• 响应：{"id":123, "name":"John", "status":true}

与 @RequestParam 区别：

特性 @PathVariable @RequestParam
绑定位置 URL 路径中的动态片段 URL 查询参数（如 ?key=value）
设计影响 明确资源层级（RESTful 风格） 不影响资源路径，用于过滤/分页

总结

• **@RequestMapping**：定义请求路径的入口。
• **@ResponseBody**：直接返回数据，适用于 API 响应。
• **@RequestBody**：接收复杂请求数据，如 JSON 对象。
• **@PathVariable**：处理 RESTful 路径参数，增强 URL 可读性。

四、中间件与数据库

  1. Redis
    • 用途：缓存热点数据、分布式锁、会话管理。
    • 数据结构：String、Hash、List、Set、SortedSet。
    • 缓存问题：
    ◦ 穿透：缓存空值或布隆过滤器。
    ◦ 雪崩：随机过期时间，集群部署。

  2. 消息队列（如Kafka、RabbitMQ）
    • 作用：解耦、异步处理、流量削峰。
    • 核心概念：生产者、消费者、Topic（Kafka）、Exchange（RabbitMQ）。

  3. Oracle vs MySQL
    • Oracle：支持高并发、复杂事务，商业数据库（收费）。
    • MySQL：开源，轻量级，适合Web应用。

五、其他技术点

  1. 反射（Reflection）
    • 用途：动态获取类信息、调用方法（如框架中通过类名创建对象）。
    • 缺点：性能较低，破坏封装性。

  2. Java内存模型
    • 堆（Heap）：所有线程共享，存储对象实例。
    • 栈（Stack）：线程私有，存储局部变量和方法调用。

  3. Linux与部署
    • 常用命令：ls, cd, grep, chmod, ps, top。
    • 部署流程：打包JAR/WAR → 上传服务器 → 启动服务（nohup java -jar）。

  4. IO流
    • 字节流：InputStream/OutputStream（处理图片、视频）。
    • 字符流：Reader/Writer（处理文本文件）。
